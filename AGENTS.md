# 项目级 Agent 规则

## 交付与提交

- 每次完成一个明确的需求任务后，如已完成与改动直接相关的验证，应自动执行 `git commit`
- 若关键验证无法执行，提交说明中应如实反映限制，不得伪称已通过
- 提交信息格式：`<type>(scope): <summary>`
- `summary` 使用中文、动词开头、长度不超过 50 字、不加句号

## 事务边界

- 所有业务事务只允许在 `store` 层开启
- `service`、`handler`、`app` 等上层不得直接开启数据库事务
- 若一个用例需要原子性，应优先新增或扩展 `store` 方法，由 `store` 统一封装事务边界

## 后端函数注释

- 所有后端代码函数都必须编写注释
- 注释至少说明：函数用途、关键入参含义、返回值含义
- 对导出函数使用 Go 风格函数注释；对未导出但有业务含义的函数也应补充简洁注释
- 注释要求简洁准确，避免空泛描述与明显重复函数名
- 注释使用中文
- 不对存量代码做纯注释式批量补齐；仅在函数被本次需求修改时顺手补注释
- 所有本次新增的后端函数，提交前必须补齐中文注释
